* 설정파일 경로 (패키지 설치 시 자동)

  /etc/bind
------------------------------------------------------------------------------------------------------------------------------------
* 로그 캐시 (패키지 설치 시 자동)

  /var/cache/bind
------------------------------------------------------------------------------------------------------------------------------------
* 실행 바이너리 (패키지 설치 시 자동)

  /usr/sbin/named
------------------------------------------------------------------------------------------------------------------------------------
* 설정 파일 위치 생성 - zone 파일

  mkdir -p /etc/bind/zone

  chown -R bind:bind /etc/bind/zone

  chmod 775 /etc/bind/zone
------------------------------------------------------------------------------------------------------------------------------------
* 설정 파일 위치 생성 - 로그 파일

  mkdir -p /logs/named

  chown -R bind:bind /logs/named
------------------------------------------------------------------------------------------------------------------------------------
* rndc-key 재 생성 (BIND(named)를 원격/로컬에서 제어할 때 사용하는 인증 키)

  rndc-confgen -a  ( /etc/bind 밑에 rndc-key 자동으로 생성 )
------------------------------------------------------------------------------------------------------------------------------------
* zone 디렉터리 생성

  mkdir -p /etc/bind/zone/test

  mkdir -p /etc/bind/zone/test2

  mkdir -p /etc/bind/zone/test3
------------------------------------------------------------------------------------------------------------------------------------
* 설정 파일

  /etc/bind/named.conf
------------------------------------------------------------------------------------------------------------------------------------
* 동기화 확인 ( Serial 번호가 동일 해야 함 )

  master 서버에서 확인 ( master 서버 IP )

  dig @1.1.1.1 test.co.kr SOA +short 

  ns.test.com. dnsadmin.test.com. 2024050901 10800 900 604800 600

  Slave 서버에서 확인 ( Slave 서버 IP )

  dig @1.1.1.2 test.co.kr SOA +short

  ns.test.com. dnsadmin.test.com. 2024050901 10800 900 604800 600
